2. 更新软件源信息

首先，apt-get 检查本地的软件包缓存数据库（/var/lib/apt/lists/）中是否包含目标软件包的相关信息。

如果需要，执行 sudo apt-get update 更新缓存数据库，确保从指定的软件源获取最新的软件包信息。

3. 检查依赖关系
  • apt-get 会从本地缓存或远程软件源中获取目标软件包的 依赖关系信息。
  • 它解析目标软件包的依赖链，确保所有依赖包都可以下载和安装。例如：
    • 如果安装 A 软件包，而它依赖于 B 和 C，则 apt-get 会同时下载和安装 B 和 C。
4. 下载软件包
  • 根据软件包的信息，apt-get 向配置的 APT 软件源服务器发送请求，下载软件包文件（.deb 格式）。
  • 下载的文件会存储在 /var/cache/apt/archives/ 目录中。
5. 校验软件包完整性
  • 下载完成后，apt-get 使用软件包的 校验和（如 SHA256 哈希值）验证下载是否完整且未被篡改。
  • 如果校验失败，会重新下载或报告错误。
6. 安装软件包
  • apt-get 使用 dpkg 工具完成 .deb 包的安装。
  • 安装过程包括以下步骤：
    1. 解压缩：解压 .deb 包，将文件写入目标系统的对应目录。
    2. 文件注册：更新文件数据库（通常 /var/lib/dpkg/），记录新安装文件的位置。
    3. 配置文件生成：运行软件包的安装脚本（如 postinst），根据系统环境生成必要的配置文件。
    4. 安装依赖：按依赖链逐步安装其他必要的软件包。
7. 配置软件包
  • 某些软件包需要交互式配置（如设置数据库密码）。
  • 配置完成后，apt-get 会根据预定义的脚本更新系统环境。
8. 更新系统状态

dpkg 与 apt 的比较

  • dpkg 是一个低级工具，专门用来安装、删除、列出和查询 .deb 包，它不会自动解决包的依赖问题。因此，如果你使用 dpkg 安装 .deb 包时遇到依赖错误，你需要手动解决这些问题。
  • apt 是一个高级包管理工具，它可以自动处理依赖关系，通常推荐使用 apt 来安装和更新软件包，因为它比 dpkg 更方便。

  • -n 或–nodename 　显示主机名。
  • -r 或–release 　显示内核版本号。
  • -s 或–sysname 　显示操作系统名称。

C 库（C Library）

是操作系统中提供的一组标准函数，用于执行各种常见的操作，比如输入输出（I/O）、内存管理、字符串操作、文件处理等。不同的 Linux 发行版可能会选择不同的 C 库来满足这些需求。最常见的 C 库有两种：glibc 和 musl。

  1. glibc (GNU C Library)

  • 简介：GNU C 库，Linux 中最常见的 C 标准库。
  • 特点：
    • 兼容性：广泛支持 POSIX 标准，功能强大，几乎所有主流 Linux 发行版（如 Ubuntu、Debian、CentOS）都使用它。
    • 功能：提供多种高级功能，如多线程、网络、内存管理、I/O 操作等。
    • 社区支持：成熟的社区和文档，广泛兼容现有的软件和库。
  • 缺点：
    • 体积较大，占用更多磁盘和内存空间。
    • 启动和执行速度相对较慢。

2. musl

  • 简介：轻量级 C 库，专为嵌入式系统和容器环境设计。
  • 特点：
    • 体积小，适用于资源受限的系统（如容器、嵌入式设备）。
    • 启动快，性能优化，适合高效、快速响应的环境。
    • 简洁实现，去除不必要的特性，专注核心功能。
  • 缺点：
    • 兼容性差，一些依赖 glibc 特性的程序可能无法运行。
    • 功能较少，不支持所有现代 Linux 特性。

3. 如何选择

  • glibc：适用于常见 Linux 发行版，如 Ubuntu、Debian、CentOS 等，提供更好的兼容性，适合大多数应用。
  • musl：适用于轻量级发行版，如 Alpine Linux，适合容器化环境、嵌入式系统及需要节省资源的场景。

2、文件系统的层次结构

Linux 文件系统采用 树形结构，以根目录 / 为起点，其余目录都是 / 的子目录，常见的目录和内容如下：

/bin （Binary）

  • 存储内容:基本的用户可执行程序，例如 ls、cp、mv、cat 等。
  • 用途:包含系统启动和用户环境中需要的基本工具。任何用户都可以使用这些工具。

/etc （Etcetera）

  • 存储内容:系统的配置文件和脚本，例如网络配置（/etc/network）、用户信息（/etc/passwd）。
  • 用途:负责系统的配置和控制。常见文件包括：
    • fstab：文件系统表。
    • hosts：主机名映射。

/lib （Libraries）

  • 存储内容:系统程序和库文件，例如共享库（.so 文件）。
  • 用途:为 /bin 和 /sbin 中的可执行程序提供运行时支持。

/home （Home Directories）

  • 存储内容:普通用户的主目录，例如 /home/user1。
  • 用途:每个用户的私人数据、配置文件和个人项目存储在此。

/mnt（Mount Points）

  • 存储内容：手动挂载的文件系统。
  • 用途:为管理员手动挂载设备或文件系统提供的临时挂载点。

/opt（Optional）

  • 存储内容: 第三方软件包。
  • 用途:一些可选的、非默认的应用程序安装目录。

/proc（Process Information）

  • 存储内容:系统内核和进程信息的虚拟文件。
  • 用途:动态生成，用于提供系统运行时的信息。例如：
    • /proc/cpuinfo：CPU 信息。
    • /proc/meminfo：内存信息。

/root（Root Home）

  • 存储内容超级用户（root）的主目录。
  • 用途:提供 root 用户的个人文件和配置。

/usr （User System Resources）

  • 存储内容:用户程序和数据。

  • 用途:包含子目录：

    • /usr/bin：非基本用户命令。
    • /usr/sbin：非基本管理员工具。
    • /usr/lib：库文件。
    • /usr/share：共享数据。

/var （Variable）

  • 存储内容:可变数据，例如日志文件、邮件队列、缓存。
  • 用途:常见子目录：
    • /var/log：日志文件。
    • /var/tmp：长期保存的临时文件。
    • /var/spool：打印队列、邮件队列等。

3、文件类型

Linux 文件系统支持以下几种主要的文件类型：

  1. 普通文件（Regular File）：文本文件、二进制文件、脚本等。
  2. 目录文件（Directory File）：用于组织文件和子目录。
  3. 链接文件（Link File）：类似快捷方式，分为硬链接和软链接。
  4. 设备文件（Device File）：字符设备（如键盘）和块设备（如硬盘）。
  5. 管道文件（Pipe File）：进程间通信的通道。
  6. 套接字文件（Socket File）：用于网络通信的接口。

    • ~/.profile
    • ~/.bash_profile

内建命令的特点

  1. 执行效率高：

    当你在终端输入一个命令时，Shell 会首先检查这个命令是不是它的内建命令。如果是，Shell 会直接在自己的内存空间里执行这个命令，不需要去硬盘上寻找外部的可执行文件，也不需要创建新的进程。这省去了文件查找、进程创建和加载程序的开销，所以执行速度通常比外部命令快很多。

  2. 与 Shell 紧密结合：

    内建命令能够直接修改或访问 Shell 的内部状态和环境。比如，cd (改变目录) 命令必须是内建的，因为它需要直接改变当前 Shell 的工作目录。如果 cd 是一个外部命令，它只能改变它自己这个新进程的工作目录，而无法影响到启动它的那个 Shell 进程。

  3. 无法通过独立路径执行：

    你无法像执行外部命令那样，通过指定一个绝对路径 (例如 /bin/ls) 来执行内建命令。因为它们没有对应的独立文件在文件系统中。

常见的内建命令

  • cd: 改变当前工作目录。
  • pwd: 显示当前工作目录。
  • echo: 在终端显示文本或变量内容。
  • export: 设置或显示环境变量。
  • source (或 . ): 在当前 Shell 环境中执行一个脚本，而不是创建一个新的子 Shell。这在加载配置文件或环境变量时非常有用。
  • alias: 创建或显示命令别名。
  • unalias: 删除命令别名。
  • history: 显示命令历史记录。
  • exit: 退出当前 Shell 会话。
  • type: 用于判断一个命令是内建命令、外部命令还是别名等。

  • 简介：Bash(Bourne Again Shell) 是最常用的 Shell，默认情况下大多数 Linux 发行版使用 Bash。它是 Bourne Shell (sh) 的改进版，提供了更多的功能和更强大的脚本支持。

2. **sh **

  • 简介：sh(Bourne Shell) 是最早的 Unix Shell，由 Stephen Bourne 在 1970s 开发。它是许多后来的 Shell 的基础。
  • 特点：
    • 基本功能强大，但功能较为简单，缺乏 Bash 的一些高级特性（如数组、函数等）。
    • 脚本语言广泛兼容，许多 Unix 系统的 /bin/sh 指向其他更强大的 Shell，如 Bash 或 Dash。

3. Zsh

  • 简介：Zsh (Z Shell) 是一个功能丰富的 Shell，提供了许多比 Bash 更强大的特性，尤其是在命令补全和交互式使用方面。

    • 高度可定制，可以自定义提示符、命令补全、插件等。
    • 支持自动纠错，拼写建议等增强功能。
    • 支持改进的通配符匹配（如 ** 支持递归目录）。
    • 强大的插件系统，例如通过 oh-my-zsh 管理插件。
  • 常用场景：适用于需要丰富交互式功能和高度定制化的用户，特别是开发人员和高级用户。

4. Fish

  • 简介：Fish (Friendly Interactive Shell) 是一种旨在提供更友好的用户体验的 Shell。它不完全兼容传统的 sh 或 bash，但有许多独特的功能和自动化工具。
  • 特点：
    • 提供开箱即用的命令补全和语法高亮。
    • 更直观的语法，避免复杂的脚本语法。
    • 自带丰富的交互式功能，如命令历史管理、自动补全等。

通过上述初始化过程，conda 命令被定义为一个 shell 函数，而不是直接调用可执行文件。

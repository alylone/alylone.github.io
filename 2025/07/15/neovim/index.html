vim的模式

Vim 拥有多种模式，最常用的有三种：

  • 普通模式 (Normal Mode)
    • 这是 Vim 的默认模式，所有快捷键命令都在此模式下生效。
    • 主要用于移动光标、删除、复制、粘贴文本等。
    • 任何时候，如果你不确定在哪个模式，按 Esc 键总能返回到普通模式。
  • 插入模式 (Insert Mode)
    • 此模式下，Vim 和普通文本编辑器一样，可以输入文字。
    • 进入方式 (从普通模式):
      • i：在当前光标前插入
      • a：在当前光标后追加
      • I：在当前行的行首插入
      • A：在当前行的行末追加
      • o：在当前行的下方新建一行并进入插入模式
      • O：在当前行的上方新建一行并进入插入模式
  • 可视化模式 (Visual Mode)
    • 用于“选中”文本块，方便进行复制、删除等批量操作。
    • 进入方式 (从普通模式):
      • v：进入字符可视化模式，可以逐字符选择。
      • V：进入行可视化模式，可以整行选择。
      • Ctrl + v：进入块可视化模式，可以进行矩形区域选择。

vim的操作

光标移动 (在普通模式下)

1.基础移动

  • h：左
  • j：下
  • k：上
  • l：右

2.单词级移动

  • w：移动到下一个单词的词首。
  • b：移动到上一个单词的词首。
  • e：移动到当前或下一个单词的词尾。

3.行内移动

  • 0：移动到当前行的行首 (第1列)。
  • ^：移动到当前行第一个非空白字符。
  • $：移动到当前行的行末。
  • f{char}：移动到当前行下一个出现的 {char} 字符上。例如 fa 会移动到下一个 ‘a’ 上。
  • t{char}：移动到当前行下一个出现的 {char} 字符的前一个位置。

4.屏幕与文件级移动

  • gg：移动到文件的第一行。
  • G：移动到文件的最后一行。
  • {n}G 或 :{n}：移动到文件的第 n 行 (例如 10G 跳转到第10行)。
  • H：移动到当前屏幕的顶部 (High)。
  • M：移动到当前屏幕的中间 (Middle)。
  • L：移动到当前屏幕的底部 (Low)。
  • Ctrl + f：向下翻一页 (Forward)。
  • Ctrl + b：向上翻一页 (Backward)。

文本编辑 (复制、删除、粘贴)

Vim 的编辑操作遵循一个强大的语法：操作符 (operator) + 动作 (motion)。例如 d 是删除操作符，w 是移动一个单词的动作，组合起来 dw 就是“删除一个单词”。

1.复制 (Yank)

  • 操作符: y
  • yy：复制当前整行。
  • yw：复制一个单词。
  • y$：从当前光标复制到行末。
  • yG：从当前行复制到文件末尾。
  • 在可视化模式(v或V)下选中内容后，按 y 复制选中部分。

2.删除 / 剪切 (Delete)

在vim里，d操作删除的字符也会放入寄存器供粘贴时使用，所以也等同于剪切操作。

  • 操作符: d (删除的内容会进入剪贴板，所以 d 也相当于剪切)
  • dd：删除当前整行。
  • dw：删除一个单词。
  • d$ 或 D：从当前光标删除到行末。
  • dG：从当前行删除到文件末尾。
  • x：删除光标下的单个字符。
  • 在可视化模式下选中内容后，按 d 删除选中部分。

3.修改 (Change)

  • 操作符: c (这是一个组合技：删除并立即进入插入模式)
  • cc 或 S：删除当前整行并进入插入模式。
  • cw：删除一个单词并进入插入模式 (Change Word)。
  • c$ 或 C：从当前光标删除到行末并进入插入模式。

4.粘贴 (Paste)

  • p：在光标后或当前行的下一行粘贴。
  • P：在光标前或当前行的上一行粘贴。

5.撤销与重做

  • u：撤销上一步操作 (Undo)。
  • Ctrl + r：重做上一步被撤销的操作 (Redo)。

跳转与搜索

1.搜索

  • / {pattern}：从当前位置向下搜索 {pattern}。
  • ? {pattern}：从当前位置向上搜索 {pattern}。
  • n：跳转到下一个搜索结果。
  • N：跳转到上一个搜索结果。

2.括号匹配

%：在成对的括号 (), [], {} 之间来回跳转。

多行移动Tab

  1. 第m行到第n行右移一个tab

    ：m,n>

  2. 第m行到第n行左移一个tab

    ：m,n<

保存与退出 (命令模式)

这些命令需要在普通模式下先输入冒号 : 进入命令模式。

  • :w：保存文件 (Write)。
  • :q：退出 (Quit)。
  • :wq：保存并退出。
  • ZZ：(在普通模式下) 保存并退出的快捷方式，等同于 :wq。
  • :q!：强制退出，不保存任何修改。
  • :w {filename}：将当前内容另存为 {filename}。
  • :wq!：强制保存并退出 (例如文件为只读时)。

当你输入 :!command_name 时，Vim/Neovim 会做以下事情：

  1. 启动一个 Shell 进程： 它会调用你系统默认的 Shell（在 Linux/macOS 上通常是 Bash, Zsh 等，在 Windows 上可能是 cmd.exe 或 PowerShell）。
  2. 将你的 command_name 作为参数传递给 Shell： Shell 会尝试在你的系统 PATH 环境变量中查找 command_name 这个可执行文件。
  3. 执行命令： 如果找到，Shell 就会执行这个命令。
  4. 捕获输出： 命令的输出（标准输出和标准错误）会被捕获并显示在 Vim 窗口的底部。

vim的内建命令

你可以直接在命令模式（按下 : 键后）输入它们，而无需通过外部 Shell。它们是 Vim 强大功能的核心，涵盖了从文件操作到文本编辑，再到窗口管理的方方面面。

这是vim自己的自建命令(vim自带的)，不依靠外部

以下是一些 Vim 最常用和最重要的内建命令类别及示例：

1. 文件操作 (File Operations)

  • :w (write): 保存当前文件。
  • :w!: 强制保存（例如，当文件是只读时）。
  • :q (quit): 退出当前窗口或 Vim。
  • :q!: 强制退出，不保存更改。
  • :wq: 保存并退出。
  • :x: 保存并退出（与 :wq 类似，但只有在文件被修改过时才写入）。
  • :e {file} (edit): 编辑另一个文件。
  • :e! {file}: 强制编辑另一个文件，放弃当前文件的未保存更改。
  • :r {file} (read): 在当前光标位置插入文件内容。
  • :sav {file} (saveas): 另存为新文件。
  • :f (file): 显示当前文件信息（文件名、行数、编码等）。
  • :bd (bdelete): 关闭当前缓冲区并从缓冲区列表中删除。

2. 文本编辑与导航 (Text Editing & Navigation)

  • :s/{old}/{new}/g (substitute): 替换当前行中所有匹配的文本。
    • : %s/{old}/{new}/g: 替换整个文件中所有匹配的文本。
    • : %s/{old}/{new}/gc: 替换整个文件，并每次替换都进行确认。
  • :d (delete): 删除当前行。
    • :{range}d: 删除指定范围的行。
  • :y (yank): 复制行（通常和视觉模式一起用）。
  • :p (put): 粘贴。
  • :j (join): 合并两行。
  • :set {option} (设置选项): 设置各种 Vim 选项。
    • :set nu: 显示行号。
    • :set nonu: 不显示行号。
    • :set ic: 忽略大小写进行搜索。
    • :set noic: 不忽略大小写。
  • :map {lhs} {rhs}: 定义键盘映射。
    • :noremap {lhs} {rhs}: 定义非递归映射。
  • :syntax on/off: 开启/关闭语法高亮。
  • :colorscheme {name}: 设置颜色方案。

3. 窗口与缓冲区 (Windows & Buffers)

  • :sp {file} (split): 水平分割窗口。
  • :vsp {file} (vsplit): 垂直分割窗口。
  • :new: 创建一个新窗口和新空缓冲区。
  • :bn (bnext): 切换到下一个缓冲区。
  • :bp (bprevious): 切换到上一个缓冲区。
  • :b {n} / :b {name}: 切换到指定编号或名称的缓冲区。
  • :ls (buffers): 列出所有缓冲区。
  • :clo (close): 关闭当前窗口。
  • :on (only): 关闭除当前窗口外的所有窗口。

4. 宏与执行 (Macros & Execution)

  • :@a: 执行寄存器 a 中的宏。
  • :source {file}: 执行 Vimscript 文件（如 .vimrc）。

5. 帮助与信息 (Help & Information)

  • :h {topic} (help): 获取帮助。这是学习 Vim 命令最重要的途径。
    • :h :w: 查看 :w 命令的帮助。
    • :h pattern: 查看关于正则表达式模式的帮助。
  • :reg (registers): 显示所有寄存器内容。
  • :marks: 显示所有标记。

6. 历史与撤销 (History & Undo)

  • :undo: 撤销上一次更改。
  • :redo: 重做上一次撤销的更改。
  • :earlier {N}m/h/d: 返回到 N 分钟/小时/天前的状态。
  • :later {N}m/h/d: 前进到 N 分钟/小时/天后的状态。
  • :history: 查看命令历史。

7. 外部命令集成 (External Command Integration)

  • :! {command}: 执行外部 Shell 命令。
  • :r! {command}: 执行外部 Shell 命令并将输出插入到当前光标位置。
  • :{range}! {command}: 过滤指定范围的文本通过外部 Shell 命令。

8. 调试与脚本 (Debugging & Scripting)

  • :debug: 启动调试模式。
  • :echomsg {message}: 在命令行显示消息。

这只是一个概览，Vim 实际的内建命令数量非常庞大，并且每个命令通常都有许多变体和修饰符。学习 Vim 的最佳方式就是利用其强大的 :help 命令。当你对某个功能或命令有疑问时，直接 ::h {你的疑问} 就能找到详细的文档。

lua

Lua (读作 “LOO-ah”) 是一种轻量级、高效、可嵌入的脚本语言。它的设计目标是作为应用程序的扩展语言，而不是一个独立的操作系统级脚本语言。

Lua 是 Neovim (nvim) 中用来配置编辑器和编写插件的主要推荐语言。

lua在neovim的工作原理:

Lua 在 Neovim 中“工作”的底层原理，可以概括为：Neovim 内嵌了一个 Lua 解释器（通常是 LuaJIT），并提供了一套 C API 封装，使得 Lua 代码能够直接与 Neovim 的核心数据结构和功能进行交互。

  1. Neovim 核心 (C Core)：一切的基础。
  2. C API (nvim_api)：暴露内部功能的标准接口。
  3. API 封装 (Wrappers)：为其他语言搭起沟通的桥梁。
  4. Lua 调用 (LuaJIT & FFI)：在 Neovim 中实现高效的脚本控制。

1. Neovim 核心 (The C Core)

  • 功能作用:

    Neovim 的最底层是其核心功能，几乎完全由 C 语言 实现。这部分包含了文本编辑器的所有基础和核心逻辑，例如：

    • 缓冲区 (Buffer) 和窗口 (Window) 管理
    • 文件 I/O 操作
    • 文本操作（插入、删除、复制、粘贴）
    • 终端模拟
    • 事件循环 (Event Loop)
    • Vimscript 的解析与执行

  • 实现语言: C (以及少量 C++)

  • 简要介绍:

    可以将 C 核心视为 Neovim 的“引擎”。它追求极致的性能和稳定性，是所有上层功能和外部插件运行的基石。为了让这个强大的引擎能被外部世界所驱动和控制，Neovim 设计了一套标准化的接口，这就是接下来要介绍的 C API。

2. C API (nvim_api)

  • 功能作用:

    C API 是 Neovim 官方定义的一套稳定、标准化、向后兼容的函数接口。它的核心使命是将 Neovim 内部的复杂功能，以一种清晰、可控的方式暴露给外部客户端（Client）或内部脚本语言使用。任何遵守这套 API 规范的程序，都可以远程或本地地控制 Neovim 编辑器。

    例如，nvim_buf_get_lines() 函数允许你获取指定缓冲区的文本行，而 nvim_open_win() 则可以创建一个新的浮动窗口。

  • 实现语言:

    API 的定义是通过一个特殊的元数据文件（api/nvim.c 中的 api_metadata 或 api_level 相关定义）来描述的。Neovim 在编译时会根据这份元数据自动生成对应的 C 函数原型和文档。

  • 相互协同:

    C API 是 C 核心与外部世界沟通的唯一官方桥梁。它扮演着防火墙和翻译官的角色：

    • 隔离复杂性: 外部调用者无需关心 Neovim 内部数据结构的具体实现，只需调用稳定的 API 即可。
    • 保证稳定性: 即使 Neovim 内部代码重构，只要 API 保持不变，现有的插件和工具就能继续工作。

    这个 API 不仅仅是为 Lua 设计的，任何语言（Python, Go, Rust, JavaScript…）都可以通过 MessagePack-RPC 协议来调用它，这也是 Neovim 强大跨平台和远程控制能力的来源。

3. C API 封装 (API Wrappers)

  • 功能作用:

    直接在 Lua 中使用原始的 C 函数既不方便也不安全。因此，需要一个“封装层”(Wrapper) 来将底层的 C API 转换为特定语言（在这里是 Lua）更容易使用的形式。

    对于 Lua 来说，这个封装层主要做了以下几件事：

    • 数据类型转换: 将 C 的数据类型（如 char*, int, Array, Dictionary）转换为 Lua 对应的类型（如 string, number, table）。
    • 错误处理: 将 C API 返回的错误码或状态，转换为 Lua 的错误机制（例如，返回 nil 和错误信息）。
    • 提供更友好的函数签名: 让 Lua 开发者能以更自然、更符合 Lua 编程习惯的方式调用 API。

  • 实现语言:

    这个过程通常是自动化的。Neovim 在构建过程中，会解析之前提到的 API 元数据，并为 Lua 自动生成对应的封装代码。这些生成的代码位于 Neovim 源码的 src/nvim/lua/vim.lua 和相关模块中。

  • 相互协同:

    封装层是连接 C API 和 Lua 脚本的中间人。它使得 Lua 开发者可以像调用普通的 Lua 函数一样，去使用 Neovim 的核心功能。

    例如，当你在 Lua 中写下 vim.api.nvim_buf_get_lines(...) 时，你并不是直接调用到了 C 语言的实现，而是调用了那个自动生成的 Lua 封装函数。这个 Lua 函数会处理好参数的传递、类型的转换，然后再通过后续的机制去执行真正的 C API。

    Neovim 的 C API 封装，就是一套对开发者更友好、更易用的 Lua 函数。

    这个封装层的核心目的，就是隐藏底层 C 语言的复杂性，让 Lua 开发者能用最自然、最符合 Lua 习惯的方式与 Neovim 的核心功能互动。

4. Lua 调用 C API (LuaJIT & FFI)

  • 功能作用:

    这是整个流程的最后一步，也是最神奇的一步。Neovim 内置了 LuaJIT(是一个针对 Lua 语言的高性能即时编译器)。LuaJIT 的一个杀手级特性是它的 FFI (Foreign Function Interface) 库。

    外部函数接口（Foreign Function Interface），简称 FFI，是一种允许以一种编程语言编写的代码调用以另一种编程语言编写的函数或服务的机制。它本质上是一个程序性的接口，充当了两种不同语言运行时（Runtime）环境之间的桥梁，使得它们能够进行直接的、高性能的互操作。

    在 LuaJIT 的语境下，FFI 特指其内置的一个强大库，该库允许 Lua 代码直接与 C 语言进行交互。

    FFI 允许 Lua 代码直接加载 C 的头文件（.h），解析 C 的函数声明和数据结构，然后直接调用动态链接库（.so 或 .dll）中的 C 函数，整个过程无需任何手写的 C 扩展或中间层代码，性能开销极低。

  • 相互协同与流程总结:

    现在，我们可以将整个调用链串联起来：

    1. 开发者编写 Lua 代码: 你在 init.lua 或插件中写下 vim.api.nvim_get_current_buf()。
    2. 调用 Lua 封装: 这行代码执行了 Neovim 自动生成的 vim.api 模块中的 nvim_get_current_buf 这个 Lua 函数。
    3. Lua 封装通过 FFI 执行:
      • 这个 Lua 封装函数内部使用了 LuaJIT 的 FFI 功能。
      • FFI 根据预先加载的 Neovim C API 函数声明 (cdef)，知道了 nvim_get_current_buf 这个 C 函数的签名（参数和返回值类型）,自动处理 Lua 类型与 C 类型之间的数据转换。
      • FFI 直接调用了 Neovim 主程序中暴露出来的 nvim_get_current_buf C 函数的内存地址。
    4. 执行 C API: C API 函数 nvim_get_current_buf 被执行。
    5. 进入 C 核心: 该 C API 函数会进一步调用 Neovim C 核心中的内部逻辑，获取当前缓冲区的句柄（一个整数）。
    6. 返回结果:
      • C 核心将结果（整数句柄）返回给 C API。
      • FFI 获取到 C 函数的返回值。
      • Lua 封装层接收到 FFI 返回的 C 数据，并将其转换为 Lua 的 number 类型。
      • 最后，这个 number 作为 vim.api.nvim_get_current_buf() 的结果返回给你的 Lua 脚本。

4.lazyvim客制化

LazyVim 的核心设计理念是“约定优于配置”，它提供了一套非常完善的默认配置，但同时也保留了极高的可扩展性。客制化的关键在于不修改 LazyVim 的核心文件，而是在你自己的配置文件中进行覆盖 (Override) 和 扩展 (Extend)。

配置目录分部

所有个人配置都应该放在 ~/.config/nvim/lua/ 目录下，主要是以下两个地方：

  1. ~/.config/nvim/lua/config/：用于存放核心设置，如按键绑定、编辑器选项等。
  2. ~/.config/nvim/lua/plugins/：用于管理所有插件，包括添加新插件、覆盖默认插件配置或禁用插件。

全局配置与插件配置是分离的。

  • init.lua: 这是 Neovim 启动时寻找的第一个配置文件。其唯一的作用就是引导启动 lazy.nvim 插件管理器
  • lua/config/ 目录: 这个目录是所有全局性、不专属于某个特定插件的配置的家。LazyVim 会在适当的时机自动加载这些文件，用户不需要手动 require 它们 。
    • options.lua: 用于设置 Neovim 的核心选项，等同于在 vimrc 中使用 :set 命令。例如 vim.opt.number = true (显示行号)。重要的是，这个文件中的选项会在 lazy.nvim 启动之前加载，确保了基础编辑环境在插件加载前就已设定好 。
    • keymaps.lua: 用于定义全局快捷键，这些快捷键不依赖于任何特定插件。例如，可以定义一个窗口切换的快捷键。这些快捷键会在 VeryLazy 事件（即 Neovim 启动完成后不久）时加载 。
    • autocmds.lua: 用于定义全局自动命令，例如在保存文件时自动格式化，或在进入某种文件类型时设置特定选项。它们同样在 VeryLazy 事件时加载 。
    • lazy.lua: 这是 lazy.nvim 插件管理器本身的核心配置文件。它负责引导 lazy.nvim，并告诉它去哪里寻找用户的插件定义（即 lua/plugins/ 目录）。
  • lua/plugins/ 目录: 所有与插件相关的配置都应放在这里。lazy.nvim 会自动扫描并加载这个目录下的所有 .lua 文件 。无论是添加新插件、修改 LazyVim 默认插件的行为，还是禁用某个不需要的插件，所有操作都通过在此目录下创建或修改.lua 文件来完成。

插件配置方法

通过定义一个被称为“插件规约”（Plugin Spec）的 Lua table 来描述一个插件的所有信息 。

键 (Key) 类型 (Type) 用途 示例
"user/repo" string 必须。插件的 GitHub 路径，例如 "folke/tokyonight.nvim"。 { "ellisonleao/gruvbox.nvim" }
enabled boolean 设为 false 可以禁用一个插件，即使用户的配置中定义了它，或者它是 LazyVim 的默认插件。 { "folke/trouble.nvim", enabled = false }
opts table 或 function 最常用。传递给插件 setup() 函数的选项。用于配置插件的行为。 { "nvim-lualine/lualine.nvim", opts = { theme = "auto" } }
keys table 或 function 定义该插件专属的快捷键。这些快捷键只有在插件加载后才会生效。 { "ggandor/leap.nvim", keys = { "s", "S" } }
config function 或 boolean 插件加载后执行的函数。通常用于调用插件的 setup()。如果设为 false，可以阻止 lazy.nvim 自动调用 setup(opts)。 config = function() require("plugin").setup() end
init function Neovim 启动时，插件加载前执行的函数。即使插件是懒加载的，init 函数也总会执行。 init = function() vim.g.my_plugin_global_var = true end
dependencies table 插件的依赖列表。lazy.nvim 会确保依赖项在主插件之前被加载。 dependencies = { "nvim-lua/plenary.nvim" }
event string 或 table 在特定自动命令事件发生时懒加载插件。例如，"BufRead"。 event = "VeryLazy"
cmd string 或 table 在执行特定命令时懒加载插件。例如，"Telescope"。 cmd = "SymbolsOutline"
ft string 或 table 在进入特定文件类型（filetype）时懒加载插件。例如，"python"。 ft = { "markdown", "python" }

查看插件参数

这里以nvim-treesitter为例

  1. 最直接的方法就是阅读 GitHub 上的README.md,绝大部分插件都会在 README.md 文件中提供一个 “Configuration” 或 “Setup” 部分

    例如，从 nvim-treesitter 的 README 中，我们可以看到这样的结构：

  • ensure_installed: 一个列表，告诉插件要确保安装哪些语言的解析器。
  • sync_install: 一个布尔值 (true/false)，决定是否同步安装。
  • highlight: 一个表 (table)，里面包含了更多关于语法高亮的子选项，比如 enable 和 disable。

这就是最核心的参数列表。你可以把这个结构复制到你自己的 LazyVim 配置中进行修改。

为什么是require('nvim-treesitter.configs').setup {}?

  • require('module.path')

    • 功能: require 是 Lua 的内置函数，用于查找、加载并执行指定的模块文件。

    • 路径: 参数 'module.path' 是一个模块路径。Neovim 会在其 runtimepath 中搜索 module/path.lua 文件。. 在此作为目录分隔符。

    • 返回值: require 函数成功执行后，会返回模块的导出(return)内容，这通常是一个包含插件 API 的 Lua 表（Table）。

  • .setup

    • 本质: . 是 Lua 中用于访问表成员的操作符。.setup 意为访问由 require 返回的表中间键为 setup 的值。

    • 约定: 按照社区约定，插件作者会提供一个名为 setup 的函数，作为插件配置和初始化的标准、统一入口点。

  • {...}

    • 语法: { key = value, ... } 是 Lua 中创建表的字面量语法。

    • 作用: 这个表包含了所有用户自定义的配置项。它将作为唯一的参数被传递给 .setup 函数，以供插件内部使用。

  • 综上：setup函数通过接收opts向插件传递具体配置、开关功能和修改默认行为。

  1. 通过help命令，:help nvim-treesitter

这会打开插件的帮助文档。文档的结构通常和 GitHub 上的类似(其实是doc/nvim-treesitter.txt，此文件就是帮助文档)，但更适合在 Vim 中阅读。

  1. 阅读源代码

如果你想知道所有的可能性，或者文档不清晰，直接看代码是最终的答案。

  1. 在 nvim-treesitter 的项目代码中，配置通常在一个名为 configs.lua 或类似的文件里。
  2. 对于 nvim-treesitter，这个文件在 lua/nvim-treesitter/configs.lua。
  3. 打开这个文件，你会看到一个 M.setup 函数。函数内部通常有一个 default_configs 表，这个表里定义了所有参数的默认值。
  4. 查看 default_configs 的结构，你就能知道所有可以被配置的参数以及它们的默认值是什么。任何在这里定义的键，你都可以在你自己的 opts 表里覆盖它。

添加新插件

当用户发现一个 LazyVim 默认未包含的优秀插件时，可以将其添加到自己的配置中。

步骤:

  1. 寻找插件: 在 GitHub 或其他代码托管平台找到插件的仓库地址，例如 ellisonleao/gruvbox.nvim。

  2. 创建配置文件: 在 lua/plugins/ 目录下创建一个文件，例如 lua/plugins/theme.lua。

  3. 编写插件规约: 在新建的文件中，返回一个包含插件规约的 table。

  4. 重启 Neovim: 下次启动 Neovim 时，lazy.nvim 会自动检测到这个新的插件规约，并提示用户进行安装。用户也可以手动运行 :Lazy sync 来同步插件。

定制（覆盖）已有插件

调整既有插件的操作，显示，按键等等

步骤:

  1. 确定目标插件: 首先需要知道要定制的插件的名称，例如 nvim-telescope/telescope.nvim。这通常可以在 LazyVim 的官方文档中找到。

  2. 创建配置文件: 在 lua/plugins/ 目录下创建一个对应的文件，例如 lua/plugins/telescope.lua。

  3. 编写覆盖规约: 在文件中添加该插件的规约，但只包含需要修改的键。lazy.nvim 会智能地将用户的规约与 LazyVim 的默认规约进行合并。

    • 合并规则: 对于 opts 这个 table，合并是递归的，用户提供的值会覆盖默认值。对于 keys, cmd, event, ft 这些列表类型的键，用户的定义会追加到默认列表之后 。

示例：修改 Telescope 的布局 假设用户不习惯 Telescope 的默认浮动窗口，希望它从顶部出现。

在这个例子中，用户只提供了 opts 中 defaults 下的部分键。lazy.nvim 会保留所有其他 Telescope 的默认选项，只用用户提供的值更新这几个特定的键。

使用函数 当需要更复杂的逻辑（例如，基于条件添加配置，或扩展一个列表）时，可以将 opts 或 keys 等键的值设置为一个函数。该函数会接收到两个参数：插件名和默认的 opts 表。

禁用插件

如果用户发现 LazyVim 的某个默认插件不符合自己的工作流，或者想用其他插件替代，可以将其完全禁用。

步骤:

  1. 确定目标插件: 例如，用户可能不使用 folke/trouble.nvim。

  2. 创建配置文件: 可以在一个统一的文件如 lua/plugins/disabled.lua 中管理所有禁用的插件，以保持清晰。

  3. 编写禁用规约: 为该插件添加一个规约，并设置 enabled = false。

1.mason.nvim

介绍:

mason.nvim 是一个 Neovim 插件，其核心功能是统一管理外部开发工具的安装、更新与卸载。这些工具主要分为四类：

  • LSP (Language Server Protocol) Servers: 提供代码分析、补全、跳转等语言智能支持的后端服务。
  • DAP (Debug Adapter Protocol) Servers: 提供代码调试功能的后端服务。
  • Linters: 用于静态代码分析，检查代码风格、错误和潜在问题的工具。
  • Formatters: 用于自动格式化代码，确保代码风格一致的工具。

从本质上讲，Mason 是一个便捷的抽象层，它将不同来源、不同安装方式的外部可执行文件，通过统一的接口集成到 Neovim 开发环境中。

使用:

所有工具的管理都汇集于 :Mason 命令提供的图形化界面中,通过:Mason进行管理，通过:MasonLog命令查看日志。
